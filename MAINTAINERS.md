# Maintainers

This project is stewarded by a core team. Maintainers own the roadmap
([docs/CHARTER.md](docs/CHARTER.md)), review changes to protected evaluation
paths, and cut releases. The authoritative code-ownership mapping lives in
[.github/CODEOWNERS](.github/CODEOWNERS); this file is the human-readable summary.

## Core maintainers

| Maintainer | GitHub | Areas |
|---|---|---|
| Ian Cruickshank | [@ianshank](https://github.com/ianshank) | All packages; owner of every protected evaluation path (see CODEOWNERS) |

## What maintainers are responsible for

- Reviewing and approving changes to protected paths (the
  `eval-change-approved` label; see [CONTRIBUTING.md](CONTRIBUTING.md#protected-paths-require-a-labeled-approval)).
- Guarding the charter scope (§3) and invariants (§4); escalation decisions (§6).
- Accepting new Architecture Decision Records (`docs/decisions/`).
- Releases and `CHANGELOG.md` curation.

## Becoming a maintainer

Sustained, high-quality contribution and review is the path. Open a discussion or
reach out to an existing maintainer; additions are made by the current core team
and recorded here and in `.github/CODEOWNERS`.

## Security

Do not report vulnerabilities to individuals here; follow [SECURITY.md](SECURITY.md).
