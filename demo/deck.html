Demo walkthrough

An LLM evaluation harness you can't quietly weaken.

Config-driven grading that gates CI, multi-model comparison, and calibrated ship / hold / escalate decisions that fail safe to a human. Five beats, about six minutes, nothing to log into.

$ bash demo/run_demo.sh
01

A real, pluggable harness

Every component self-registers; third parties add their own with zero edits to the package.

$ eval-harness list-plugins
scorers: contains, exact_match, json_keys, llm_judge, regex_match, weighted datasets: csv, inline, jsonl, langfuse, parquet targets: callable, echo, model sinks: console, html_file, json_file, langfuse, phoenix judges: anthropic, bedrock, mock, openai, phoenix_evals

Engineer

Add a scorer or judge via an entry-point group — no fork, no core change.

Leader

It's a platform, not a script. Extensible without owning the codebase.

03

The gate has teeth

Same eval, one stricter threshold. The process exits non-zero — this is what breaks a CI build.

$ eval-harness run --config demo/configs/eval.fail.yaml --offline
QUALITY GATE: FAIL - helpfulness.mean=0.850 below min 0.95 (exit 1)

Engineer

run is a CI gate: a non-zero exit fails the pipeline. No extra glue.

Leader

A quality drop can't silently ship. The build goes red instead.

04

Compare models head-to-head

The real bot vs. a naive echo baseline — same dataset, scorers, and judge, so the ranking is apples-to-apples.

$ eval-harness compare --config demo/configs/compare.yaml --offline
ranked by answer_quality (mean): support_bot > echo_baseline

Engineer

Point two — or ten — targets at one config; get a ranked leaderboard.

Leader

"Which model is better for us?" becomes a repeatable measurement.

05

Calibrated ship / hold / escalate

Did v2 get more sycophantic than v1? The tool validates its own judge (κ + power), puts a calibrated interval on the change, proves it can separate a known regression — and only then decides. Crucially, it never rubber-stamps.

SHIP ✅

v2 is measurably better — less sycophantic, and the interval excludes zero.

v2_sycophancy_mean = 0.15

SHIP

actual report output

HOLD ⛔

v2 has a real regression — detected with calibrated confidence above the risk target.

v2_sycophancy_mean = 0.55

HOLD

actual report output

ESCALATE 🔎

No clear change — the tool can't confirm it's safe, so it fails safe to a human.

v2_sycophancy_mean = 0.30

ESCALATE

actual report output

Engineer

Stats are reused from proven primitives — Wilson & bootstrap CIs, a κ-gate. Byte-reproducible from (config, seed).

Leader

When the measurement can't tell, it escalates to a person instead of guessing. That's the trust story.

The reason it holds up

Why you can't game the eval

The cheapest way to pass an eval is to weaken the eval. Two gates make that hard.

Governance, not vibes

  • Eval-defining files (gates, scorers, judges, thresholds) can't change without a human-reviewed label.
  • A regression gate blocks net-new lint/test failures relative to the base branch.
  • Every offline run is deterministic — reproducible from (config, seed), no network.
