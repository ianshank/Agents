# Support

Thanks for using this project. Here is where to go for help.

## Before asking

- **Install / run / test:** [README.md](README.md).
- **How the pieces fit / which doc to read:** [AGENTS.md](AGENTS.md) and the
  documentation index at [docs/README.md](docs/README.md).
- **Why something is designed the way it is:** the Architecture Decision Records
  under [docs/decisions/](docs/decisions/README.md).
- **A repeatable end-to-end demo:** [demo/README.md](demo/README.md).

## Questions and discussion

- Open a **GitHub Discussion** (if enabled) or a **question issue**. Include what
  you tried, the command you ran, and the output.

## Bug reports and feature requests

- File a **GitHub Issue** — the issue templates (bug report, feature request)
  will prompt you for the details we need. For bugs, include a minimal reproduction, the
  package and version, your Python version (3.10–3.12 are supported), and whether
  you were running the offline suite or a live integration.

## Security

Do **not** use public issues for vulnerabilities — follow
[SECURITY.md](SECURITY.md).

## Contributing

Want to fix it yourself? See [CONTRIBUTING.md](CONTRIBUTING.md).
