# scripts/

Operational tooling for the monorepo — feature validators, CI guards, and the
merge-gate seeding pipeline. These are **not** part of any shipped package; they
run in CI and locally. They carry their own quality gate: `ruff` + `mypy scripts`
+ `pytest --cov=scripts` at a **≥85%** floor (`scripts/.coveragerc`, feature
`F-031`).

> `scripts/validations/` and any new `tests/**` are **protected paths** — changes
> require the `eval-change-approved` label.

## Index

| Script | Purpose |
|---|---|
| `quality-gate.sh` | the committed lint + format-check + mypy + `pytest --cov` gate (generated by the `quality-gate` skill) |
| `init.py` | cross-platform project init (venv + editable install) |
| `validate.py` | spec-driven project validation (drives `features.yaml`) |
| `validate_skill.py` | skill structural + behavioral validation (canonical copy; vendored into each skill, drift-guarded) |
| `select_next.py` | feature-priority selector |
| `skill_marketplace.py` | skill registry CLI (validate / verify / list) |
| `regression_gate.py` | net-new lint/test diff vs an isolated `git worktree` baseline (`F-006`) |
| `eval_protected_paths.py` | **single source of truth** for protected eval-defining paths |
| `check_protected_changes.py` | CI guard: flags protected changes lacking the approval label (`F-007`) |
| `check_skill_script_drift.py` | CI guard: vendored skill scripts == canonical copy |
| `check_charter_drift.py` | CI guard: `docs/CHARTER.md` scope/invariants haven't silently drifted |
| `check_size_budget.py` | structural size-budget gate — file length ≤ 500 lines, etc. (ADR 0019; `.py` only) |
| `merge_gate_context.py`, `agent_confidence.py`, `_config.py` | calibrated merge-gate seeding (`F-042`) |
| `record_audit_verdict.py`, `audit_issue_sync.py` | merge-gate human-audit surface |
| `fix_loop.py` | auto-fix loop scaffolding — **design-only, disabled** (ADR 0004) |
| `run_all_e2e.ps1` | one-command e2e / user-journey harness → `artifacts/e2e-report/` |
| `migrations/` | one-off reversible data migrations |
| `validations/` | per-feature validation scripts (`F_0NN.py`) — **protected** |
| `e2e_shims/` | `sitecustomize.py` neutralizing a Windows platform-WMI hang for pytest |

## Run the gate

```bash
./scripts/quality-gate.sh all          # from the repo root
make check                             # same, via the generated Makefile
```
